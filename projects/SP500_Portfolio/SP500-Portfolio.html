0. Executive Summary

This project evaluates the effectiveness of a portfolio optimization strategy based on Modern Portfolio Theory using historical S&P 500 stock data. A walk-forward validation framework is applied to construct and rebalance portfolios over a five-year period, explicitly avoiding look-ahead bias and enabling realistic performance assessment.

Results indicate that the optimized portfolio achieves improved risk-adjusted returns compared to a passive S&P 500 benchmark, primarily through reduced volatility and more efficient asset allocation. However, performance is highly sensitive to changes in market conditions, with periods of regime shifts leading to a deterioration in relative returns.

1. Introduction and Context

Background: Investors often seek to optimize their portfolios to maximize returns while minimizing risk. The S&P500 index, comprising 500 of the largest publicly traded companies in the U.S., provides a diverse set of investment opportunities. This project aims to analyze historical stock data from the S&P500 to construct an optimized investment strategy. Problem Statement: How can we optimally allocate investments across the largest companies within the S&P500 stocks to achieve an optimal balance between risk and return?

2. Project Objective

The objective of this project is to design, evaluate, and validate a strategy for optimizing portfolios using the S&P 500. This strategy will be compared to traditional benchmarks over the past several years using a walk-forward simulation.

Specific Objectives: * Data Retrieval: Fetch historical price data for selected stocks and the S&P 500 index (SPY) using financial APIs. * Performance Analysis: Calculate key metrics such as Annualized Return, Volatility (Standard Deviation), and Beta. * Portfolio Optimization: Use Modern Portfolio Theory (Markowitz) to find the “Efficient Frontier” and determine optimal asset allocation. * Benchmark Comparison: Visualize the performance of the optimized portfolio versus the S&P 500 benchmark over the last 5 years.

3. Methodology

To achieve these objectives, the following workflow will be followed: 1. Data Acquisition: Using yfinance to download Adjusted Close prices. 2. Exploratory Data Analysis (EDA): Visualizing cumulative returns and correlation matrices to understand asset relationships. 3. Quantitative Analysis: Computing daily log returns, covariance matrices, and portfolio variance. 4. Simulation & Optimization: Running a Monte Carlo simulation (or using scipy.optimize) to identify the portfolio weights that maximize the Sharpe Ratio.

4. Tools and Libraries

This project utilizes Python for financial analysis, specifically: * yfinance: For downloading market data. * pandas & numpy: For data manipulation and vector calculations. * matplotlib & seaborn: For financial plotting and correlation heatmaps. * scipy: For portfolio optimization routines.

5.1 UNIVERSE DEFINITION (4 PERIODS: 2023 - 2026)

For each year, the universe was defined using the top 15 S&P 500 constituents as of the beginning of the period. Historical constituent lists were obtained by accessing archived versions of https://www.slickcharts.com/sp500 through the Wayback Machine (https://web.archive.org/).

To mitigate look-ahead bias and survivorship bias, the universe selection was performed dynamically. Instead of relying on current index constituents, historical snapshots of the S&P 500 rankings were retrieved for each rebalancing date. This approach ensures that the model exclusively uses information that was available at the time of portfolio formation, thereby preserving realistic historical conditions and explicitly accounting for companies that may have later fallen in ranking or exited the index.

5.2 Initialization parameters

For the walk-forward simulation, the following parameters were established:

  • Initial Capital: $10,000 USD
  • Rebalancing Frequency: Annually (at the start of each year).
  • Lookback Period: 1 year of historical data prior to each rebalancing date.

Market parameters: * Risk-free Rate: 4% (if dynamic retrieval fails). This rate is used in the Sharpe Ratio calculation.

Constraints for optimization: * Minimun weights per asset: 0% to ensure no negative share. * Maximum weights per asset: 25% to avoid optimizer get concetrate capital in a specific asset.

def get_risk_free_rate(target_date_str):
    """
    Fetches the 10-Year US Treasury Yield (^TNX) closest to the given date.
    Returns the value as a decimal (e.g., 0.04 for 4%).
    """
    ticker = "^TNX"
    end_date = pd.to_datetime(target_date_str)
    start_date = end_date - timedelta(days=15) # Look back 2 weeks to find a trading day
    
    try:
        # Added auto_adjust=True to fix warning
        rf_data = yf.download(ticker, start=start_date, end=end_date, progress=False, auto_adjust=True)['Close']
        
        if rf_data.empty:
            print(f"     [!] Warning: Could not fetch ^TNX. Defaulting to" + f" {risk_free_rate_default*100}%.")
            return risk_free_rate_default
            
        # Yahoo returns index values (e.g., 3.85), so we divide by 100
        latest_rate = rf_data.iloc[-1] / 100
        return float(latest_rate.iloc[0])
        
    except Exception as e:
        print(f"     [!] Error fetching Risk-Free Rate: {e}. Defaulting to" + f" {risk_free_rate_default*100}%.")
        return risk_free_rate_default

6. Optimization Engine

Code 
def get_optimal_weights(tickers, rebalance_date):
    end_train = pd.to_datetime(rebalance_date)
    start_train = end_train - timedelta(days=lookback_days)
    
    print(f"   > Training model ({start_train.date()} -> {end_train.date()})...")
    
    # Get Dynamic Risk Free Rate
    current_rf_rate = get_risk_free_rate(rebalance_date)
    print(f"   > Risk-Free Rate for this period: {current_rf_rate:.2%}")

    # Download training data
    try:
        # Added auto_adjust=True to fix warning and ensure adjusted prices
        data = yf.download(tickers, start=start_train, end=end_train, progress=False, auto_adjust=True)['Close']
    except Exception as e:
        print(f"     Error downloading data: {e}")
        return pd.Series([1/len(tickers)]*len(tickers), index=tickers)

    # Data Cleaning
    data = data.ffill().dropna(axis=1, how='any') # Drop stocks with any missing data
    valid_tickers = data.columns.tolist()
    
    if len(valid_tickers) == 0: return pd.Series()

    returns = data.pct_change().dropna()
    mean_returns = returns.mean() * 252
    cov_matrix = returns.cov() * 252
    num_assets = len(valid_tickers)
    
    # Objective Function: Minimize Negative Sharpe Ratio
    def negative_sharpe(weights):
        p_ret = np.dot(weights, mean_returns)
        p_vol = np.sqrt(np.dot(weights.T, np.dot(cov_matrix, weights)))
        return -(p_ret - current_rf_rate) / p_vol

    # Constraints and Bounds Setup
    # 1. Standard Constraint: Sum of weights must equal 1 (100% allocation)
    constraints = [{'type': 'eq', 'fun': lambda x: np.sum(x) - 1}]

    # 2. Group Constraints: Handling Dual-Class Stocks (e.g., GOOG + GOOGL)
    # The SUM of these specific tickers to respect the max_weight limit.
    
    for group_name, group_tickers in restricted_groups.items():
        # Find the numerical index for each ticker in the group (if they exist in valid data)
        group_indices = [valid_tickers.index(t) for t in group_tickers if t in valid_tickers]
        
        if len(group_indices) > 0:
            # Logic: max_weight - (sum of group weights) >= 0
            constraints.append({
                'type': 'ineq', 
                'fun': lambda x, idx=group_indices: max_weight - np.sum([x[i] for i in idx])
            })
            print(f"     -> Group constraint applied to {group_name}: {[valid_tickers[i] for i in group_indices]}")

    #3. Individual Bounds (e.g 0% to 25% per single ticker)
    bounds = tuple((min_weight, max_weight) for _ in range(num_assets)) 
    
    try:
        result = minimize(negative_sharpe, num_assets * [1./num_assets], 
                          method='SLSQP', bounds=bounds, constraints=constraints)
        return pd.Series(result.x, index=valid_tickers)
    except:
        print("     Optimization failed, defaulting to Equal Weights.")
        return pd.Series([1./num_assets]*num_assets, index=valid_tickers)

7.1 Portfolio Rebalancing Simulation

Code 
# --- Initialization of existing variables ---
portfolio_value = [initial_capital]
sorted_dates = sorted(list(rebalance_map.keys()))
portfolio_dates = [pd.to_datetime(sorted_dates[0])]

# List to store allocation history
allocation_history = [] 

print(f"=== STARTING WALK-FORWARD BACKTEST ===")

for i in range(len(sorted_dates)):
    current_date = sorted_dates[i]
    tickers = rebalance_map[current_date]
    
    if i < len(sorted_dates) - 1:
        next_date = sorted_dates[i + 1]
    else:
        next_date = datetime.now().strftime('%Y-%m-%d')
    
    if pd.to_datetime(current_date) > datetime.now():
        break

    print(f"\n--- REBALANCE DATE: {current_date} ---")
    
    # A. Optimal Weights
    weights = get_optimal_weights(tickers, current_date)
    
    # Convert the weights Series into a plotting-friendly format
    weights_df = weights.to_frame(name='Weight')
    weights_df['Ticker'] = weights_df.index
    weights_df['Rebalance Date'] = current_date
    allocation_history.append(weights_df)
    
    # B. Simulation
    print(f"   > Simulating returns until {next_date}...")
    
    price_data = yf.download(
        weights.index.tolist(),
        start=current_date,
        end=next_date,
        progress=False,
        auto_adjust=True
    )['Close']
    
    if not price_data.empty:
        period_returns = price_data.pct_change().dropna()
        
        available_tickers = period_returns.columns.intersection(weights.index)
        active_weights = weights[available_tickers]
        active_weights = active_weights / active_weights.sum()
        
        daily_portfolio_return = period_returns[available_tickers].dot(active_weights)
        
        current_balance = portfolio_value[-1]
        cumulative_period = (1 + daily_portfolio_return).cumprod() * current_balance
        
        portfolio_value.extend(cumulative_period.values)
        portfolio_dates.extend(cumulative_period.index)
    else:
        print("     Not enough data for this period.")
