Professional S&P 500 Portfolio Optimization Framework

Project Overview

This framework implements a robust, modular system for Walk-Forward Portfolio Optimization. It goes beyond simple static backtests by simulating real-world constraints, transaction costs, and multiple quantitative strategies.

Key Technical Features:

  • Multi-Strategy Engine: Support for Markowitz (Sharpe), Risk Parity, and the newly implemented Sortino Ratio Optimization (maximizing return per unit of downside risk).
  • Dynamic Rebalancing: A volatility-triggered rebalancing engine that adapts to market regimes using rolling 21-day volatility thresholds.
  • SQL Data Architecture: A structured SQLite backend for managing asset metadata, historical candidates, and portfolio tracking.
  • Transaction Cost Modeling: Realistic 0.1% (configurable) fee application on every rebalance.

2. Walk-Forward Simulation Strategy Comparison

We will now run three distinct quantitative strategies and compare them against the S&P 500 (SPY) benchmark: 1. Markowitz (Sharpe): Optimizes for the highest return relative to total volatility. 2. Sortino: Optimizes for the highest return relative to downside volatility (newly added). 3. Dynamic Rebalance: Uses Sharpe optimization but triggers additional rebalances if rolling volatility exceeds 20%.

results = {}
configs = [
    {'name': 'Sharpe (Markowitz)', 'optimizer': 'markowitz', 'dynamic': None},
    {'name': 'Sortino (Downside Focus)', 'optimizer': 'sortino', 'dynamic': None},
    {'name': 'Dynamic Sharpe (20% Vol Trigger)', 'optimizer': 'markowitz', 'dynamic': 0.20}
]

for cfg in configs:
    bt = WalkForwardBacktester(
        optimizer_type=cfg['optimizer'], 
        vol_threshold=cfg['dynamic']
    )
    # Use the local file since we are running inside the project folder
    equity, _ = bt.run('portfolio_historical_candidates.csv')
    results[cfg['name']] = equity

# Fetch Benchmark for comparison
dm = DataManager()
spy_prices = dm.fetch_prices(['SPY'], results['Sharpe (Markowitz)'].index[0], results['Sharpe (Markowitz)'].index[-1])
spy_norm = (spy_prices / spy_prices.iloc[0]) * 10000

4. Risk-Adjusted Metrics

Numbers reveal the true quality of a strategy. We calculate the Sharpe Ratio, Sortino Ratio, and Maximum Drawdown for each approach.

metrics_list = []
for name, equity in results.items():
    m = WalkForwardBacktester.calculate_metrics(equity, results['S&P 500 (SPY)'])
    m['Strategy'] = name
    # Ensure all metric values are scalars (floats)
    for key in ['Total Return', 'Sharpe Ratio', 'Sortino Ratio', 'Max Drawdown', 'Volatility']:
        val = m[key]
        if hasattr(val, 'iloc'):
            m[key] = float(val.iloc[0])
        else:
            m[key] = float(val)
    metrics_list.append(m)

metrics_df = pd.DataFrame(metrics_list).set_index('Strategy')
metrics_df[['Total Return', 'Sharpe Ratio', 'Sortino Ratio', 'Max Drawdown', 'Volatility']].style.format({
    'Total Return': '{:.2%}',
    'Sharpe Ratio': '{:.2f}',
    'Sortino Ratio': '{:.2f}',
    'Max Drawdown': '{:.2%}',

})

6. Conclusions & Strategic Insights

Based on the walk-forward simulation results and the visual evidence from our comparative analysis, we can draw the following conclusions:

1. Robust Alpha Generation

All quantitative optimization strategies significantly outperformed the S&P 500 (SPY) benchmark. While the benchmark returned 82.76% over the simulation period, our models delivered returns ranging from 108.26% to 130.24%, demonstrating the value of active, walk-forward rebalancing even after accounting for transaction costs.

2. The Power of Downside Focus (Sortino Optimization)

The Sortino (Downside Focus) strategy was the top performer in terms of absolute return (130.24%). By specifically penalizing “bad” (downside) volatility rather than all volatility, the optimizer was able to capture more of the market’s upside while maintaining a maximum drawdown (-18.78%) comparable to the benchmark, but with far superior wealth accumulation.

3. Risk-Return Optimization with Dynamic Rebalancing

The Dynamic Sharpe strategy successfully reduced portfolio volatility to 18.73% (the lowest among active strategies) compared to the pure Markowitz approach (20.62%). However, the frequent rebalancing triggered by the 20% volatility threshold introduced transaction costs and potential “whipsaw” effects, leading to a lower total return (108.26%). This highlights the classic trade-off between absolute stability and capital growth.

4. Allocation Quality

As seen in the Sortino Allocation chart, the optimizer demonstrates high conviction in high-quality, asymmetric assets. In the most recent rebalance, the model maxed out the 25% individual asset cap for defensive leaders like WMT and GOOG. This concentration into assets with favorable downside profiles is a core driver of the strategy’s persistent outperformance across the walk-forward windows.
